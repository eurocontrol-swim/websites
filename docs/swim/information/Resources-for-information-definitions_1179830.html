Resources for information definitions

Last updated:  JANUARY 14th, 2020

This section of the handbook contains resources to help you satisfy the requirements.

  • Example information definition — This page gives an example information definition for illustrative purposes.
  • Further guidance on fulfilling the requirements
    • Recording metadata required by the specification — This guidance shows how the "metadata" requirements found in the SWIM Information Definition Specification can be satisfied using the structures found in the XML Schema Definition language and based on common metadata standards. 
    • Detailing the concepts using XSD — This guidance shows how requirements concerning the concepts found in an information definition can be satisfied using the structures found in the XML Schema Definition language.
    • Understanding and recording mappings — This guidance is provided in order to allow you to better understand mappings from an information definition to the AIRM and how to record them in the information definition. 
  • Verification checklist
  • Using the AIRM — This section gives a very quick overview of the AIRM in order to help those creating the semantic correspondences. It is in no way an exhaustive tutorial on the AIRM.
